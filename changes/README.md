# Changes Directory

This directory contains change documentation for each PR/feature.

## File Naming

```
YYYY-MM-DD-HHMM-brief-description.md
```

Example: `2025-01-26-1430-add-speech-recognition.md`

## Template

```markdown
# [Brief Title]

## Why

[Why this change was made - the problem or feature request]

## How

[How it was implemented - key technical decisions]

## Key Decisions

- [Decision 1 and rationale]
- [Decision 2 and rationale]

## Testing

[How this was tested, test attestation]

## What's Next

[Future work this enables or follow-up tasks]
```

## Guidelines

- One file per PR
- Focus on the "why" not just the "what"
- Include test attestation
- Document architectural decisions
- Link to related issues/PRs
